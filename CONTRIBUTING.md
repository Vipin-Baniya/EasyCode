# Contributing to Project Core

Thank you for your interest in contributing to Project Core!

## Development Setup

1. Fork the repository
2. Clone your fork
3. Run `./scripts/setup.sh`
4. Create a branch for your changes

## Code Style

### Python
- Follow PEP 8
- Use type hints
- Add docstrings
- Run `black` and `isort`

### TypeScript
- Follow ESLint rules
- Use TypeScript types
- Document complex logic

## Testing

- Write tests for new features
- Ensure all tests pass: `./scripts/test.sh`
- Add integration tests for critical paths

## Pull Request Process

1. Update documentation
2. Add tests
3. Ensure CI passes
4. Request review
5. Address feedback

## Commit Messages

Use conventional commits:
- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation
- `test:` Tests
- `refactor:` Code refactoring

## Questions?

Open an issue or start a discussion.
